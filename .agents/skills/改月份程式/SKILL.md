# 批次修改白板月份數字技能 (改月份程式)

本技能可用於自動化批量修改工地或呈報照片中的白板日期月份數字。透過影像定位與融合技術，在保留原始白板背景、光影及字體質感的同時，無痕替換目標數字。

## 適用場景
當需要將某個資料夾下多張白板照片中的特定月份數字（例如「1月」），無痕替換為指定的另一個月份數字（例如「5月」），且手上有該目標數字的透明 PNG 模板或含有該數字透明圖層的 PSD 原始檔。

## 技能資源
本技能附帶一個高度參數化的 Python 腳本，放置於技能目錄下：
* `modify_month.py`：主執行程式。

## 參數與變數說明
呼叫本程式時，可透過命令列參數傳入以下變數：

| 參數 | 必要 | 說明 | 範例 / 預設值 |
|---|---|---|---|
| `--folder` | **是** | 包含待修改 JPG 圖片的資料夾路徑 | `case001\test_004-頂崁里-廢棄物清除呈報-115.05.13` |
| `--template` | **是** | 目標數字的透明模板路徑（支援 `.png` 或 `.psd`） | `0513_layer1.png` 或 `0513.psd` |
| `--psd-layer` | 否 | 若模板為 PSD，指定要擷取的透明圖層索引 | 預設為 `1` |
| `--ref-image` | 否 | 包含基準年份與原月份範本的參考圖片路徑 | 預設自動搜尋資料夾中含有 `_41.jpg` 的檔案 |
| `--year-bbox` | 否 | 年份模板在參考圖片中的 ymin,ymax,xmin,xmax 座標 | 預設為 `595,625,195,270` (適用 115. 年份錨點) |
| `--month-bbox` | 否 | 月份模板在參考圖片中的 ymin,ymax,xmin,xmax 座標 | 預設為 `599,621,276,290` (適用月份 1 的位置) |
| `--opacity` | 否 | 當停用自動明暗度偵測時，目標數字覆蓋時的固定不透明度 | 預設為 `0.62` |
| `--year-thresh` | 否 | 年份比對的信心度門檻 (0.0 ~ 1.0) | 預設為 `0.70` |
| `--month-thresh` | 否 | 月份比對的信心度門檻 (0.0 ~ 1.0) | 預設為 `0.75` |
| `--no-auto-opacity` | 否 | 停用自動明暗度匹配功能，強制使用 `--opacity` 設定的固定不透明度 | 預設為啟用自動偵測 |

## 執行步驟說明

### 步驟 1：準備環境與相依套件
執行此腳本前，需確保系統已安裝以下 Python 套件：
```bash
pip install opencv-python numpy psd-tools
```

### 步驟 2：執行批次替換命令

#### 範例 A：使用透明 PNG 模板作為輸入
若已經有透明 PNG 數字模板（例如 `0513_layer1.png`）：
```bash
python .agents/skills/改月份程式/modify_month.py --folder "case001\test_004-頂崁里-廢棄物清除呈報-115.05.13" --template "case001\test_004-頂崁里-廢棄物清除呈報-115.05.13\0513_layer1.png"
```

#### 範例 B：使用 Photoshop (PSD) 檔案作為輸入
若直接使用包含圖層的 PSD 檔案（例如 `0513.psd`），程式會自動擷取指定圖層（預設為第 1 圖層）的透明影像：
```bash
python .agents/skills/改月份程式/modify_month.py --folder "case001\test_004-頂崁里-廢棄物清除呈報-115.05.13" --template "case001\test_004-頂崁里-廢棄物清除呈報-115.05.13\0513.psd" --psd-layer 1
```

## 注意事項與調優建議
1. **路徑中文支援**：本腳本採用 UTF-8 編碼安全讀寫，完全支援包含中文或特殊字元的 Windows 資料夾路徑。
2. **多尺度定位**：因為每張照片拍攝時的距離、角度可能不同，程式會自動以 0.5 到 2.0 倍率縮放進行年份搜尋，再動態依比例推算月份搜尋區。
3. **消除原始數字**：腳本利用 `cv2.inpaint` 自動進行 Otsu 門檻偵測並搭配 2 次膨脹（Dilation），能 100% 乾淨抹除原本的數字筆跡。
4. **自動重心對齊**：程式採用「筆跡重心對齊」技術，會自動計算舊手寫字與新數字模板的墨水中心並重合，因此能容忍不同尺寸的模板。
5. **明暗度自動調和 (Auto-Opacity)**：程式預設會自動掃描每張圖片的白板手寫年份筆跡，並精準計算出該張照片的「墨水深淺與對比度」，從而動態調整貼上去的新數字明暗度（不透明度），使每張照片的字體在不同的光影、曝光與陰影下皆能達到 100% 的自然融入效果。如要關閉此功能，只需加上 `--no-auto-opacity` 即可。

## 透明數字模板（PSD/PNG）製作與像素尺寸規範

為了確保每一次批次修改都能達到 100% 的精準度與完美對齊，建議在製作 `.psd` 或 `.png` 的目標數字模板時遵循以下規範：

### 1. 緊密裁切（無外框留白）— **最重要！**
* **規範**：請務必將畫布/圖層範圍**緊密裁切（Tight Crop）**至數字筆跡的邊緣，周圍**不要留任何透明空白或安全邊距**。
* **原因**：對齊演算法是以模板的「筆跡重心」為基礎。如果周圍有不對稱的留白，會影響重心的計算，導致置中偏差。

### 2. 建議像素尺寸
* **高度**：建議在 **20 至 30 像素（pixels）** 之間。
  * 例如：同慶里的 `6` 尺寸為 `20x12`，頂崁里的 `5` 尺寸為 `21x11`。
* **高解析度相容性**：如果製作了較大解析度的數字（例如 `100x60` 像素），程式仍然能夠運作（會自動依比例縮小），但**緊密裁切**的原則依然不變。

### 3. 透明度與圖層格式
* **背景**：必須是 100% 完全透明（Alpha = 0）。
* **筆跡**：僅保留黑白板筆的墨水筆劃，且墨水部分應有適當的半透明邊緣抗鋸齒（Anti-aliasing），這樣疊合時才會呈現自然的墨水質感。
* **PSD 儲存格式**：如果使用 PSD 檔案，建議將透明數字放在**第 1 圖層（Layer 1）**（即背景層之上的第一個獨立圖層），以便腳本自動提取。
